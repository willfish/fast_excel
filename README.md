# Uber Fast Excel

`uber_fast_excel` is a maintained Ruby wrapper around
[libxlsxwriter](https://github.com/jmcnamara/libxlsxwriter) for generating XLSX
workbooks quickly through FFI.

The gem keeps the existing Ruby API and namespace:

```ruby
require "fast_excel"
```

It also supports the gem-name require path:

```ruby
require "uber_fast_excel"
```

## Installation

```ruby
# Gemfile
gem "uber_fast_excel", require: "fast_excel"
```

Or install it directly:

```sh
gem install uber_fast_excel
```

## Migration From fast_excel

Replace the gem dependency, then keep application code as-is:

```diff
- gem "fast_excel"
+ gem "uber_fast_excel", require: "fast_excel"
```

The public namespace remains `FastExcel`, so existing calls such as
`FastExcel.open`, `FastExcel::Formula`, and `FastExcel::URL` continue to work.

## Basic Usage

```ruby
require "fast_excel"

workbook = FastExcel.open("hello_world.xlsx", constant_memory: true)
workbook.default_format.set(
  font_size: 0,
  font_family: "Arial"
)

worksheet = workbook.add_worksheet("Example Report")

bold = workbook.bold_format
worksheet.set_column(0, 0, FastExcel::DEF_COL_WIDTH, bold)

price = workbook.number_format("#,##0.00")
worksheet.set_column(1, 1, 20, price)

date_format = workbook.number_format("[$-409]m/d/yy h:mm AM/PM;@")
worksheet.set_column(2, 2, 20, date_format)

worksheet.append_row(["message", "price", "date"], bold)

1.upto(1000) do
  worksheet.append_row(["Hello", (rand * 10_000_000).round(2), Time.now])
end

worksheet.append_row(["Sum", FastExcel::Formula.new("SUM(B2:B1001)")], bold)

workbook.close
```

More runnable examples live in [`examples/`](examples/).

## Workbooks

Create a temporary workbook and stream the generated bytes:

```ruby
workbook = FastExcel.open
worksheet = workbook.add_worksheet
worksheet.append_row(["Code", "Status"])

send_data(workbook.read_string, filename: "table.xlsx")
```

Create a workbook at a specific path:

```ruby
workbook = FastExcel.open("report.xlsx")
```

Use constant memory mode for large exports:

```ruby
workbook = FastExcel.open("large-report.xlsx", constant_memory: true)
```

In constant memory mode, rows are flushed as they are written and earlier rows
cannot be changed.

## Writing Values

`write_value` detects numbers, dates, times, formulas, URLs, booleans, rich
strings, and plain strings:

```ruby
workbook = FastExcel.open
worksheet = workbook.add_worksheet

worksheet.write_number(0, 0, 123_456, nil)
worksheet.write_value(1, 0, Time.now)
worksheet.write_row(2, ["strong", 123_456, Time.now])
worksheet.append_row(["bottom", 456])
worksheet << ["shortcut", 789]
```

### Dates

Excel stores dates as day numbers. `FastExcel.date_num` exposes the conversion
when you want to write the numeric value yourself:

```ruby
date_format = workbook.number_format("[$-409]m/d/yy hh:mm;@")
worksheet.write_number(0, 0, FastExcel.date_num(Time.now), date_format)
```

### Formulas

```ruby
worksheet << [1, 2, 3, 4]
worksheet << [FastExcel::Formula.new("SUM(A1:D1)")]
```

### URLs

```ruby
url_format = workbook.add_format(underline: :underline_single, font_color: :blue)
worksheet.append_row([
  FastExcel::URL.new("https://github.com/willfish/fast_excel"),
  FastExcel::URL.new("postgres://localhost")
], url_format)
```

### Rich Strings

Rich strings allow multiple formats in a single cell:

```ruby
bold = workbook.add_format(bold: true)
rich_string = FastExcel::RichString.new([
  { text: "plain " },
  { text: "bold", format: bold }
])

worksheet.write_value(0, 0, rich_string)
```

## Formatting

```ruby
format = workbook.add_format(
  bold: true,
  italic: true,
  text_wrap: true,
  font_size: 13,
  font_name: "Arial",
  font_color: :orange,
  rotation: 10,
  underline: :underline_single,
  border: :border_thin,
  bottom_color: :alice_blue,
  align: { h: :align_center, v: :align_vertical_center },
  num_format: "#,##0.00"
)
```

Shortcuts:

```ruby
workbook.bold_format
workbook.number_format("[$-409]m/d/yy h:mm AM/PM;@")
```

## Worksheet Helpers

Set column width:

```ruby
worksheet.set_column(start_col, end_col, width = nil, format = nil)
worksheet.set_column_width(col, width)
worksheet.set_columns_width(start_col, end_col, width)
```

Set row height:

```ruby
worksheet.set_row(row_num = 0, height = 30, format = nil)
```

Enable filters:

```ruby
worksheet.enable_filters!(start_col: 0, end_col: 3)
```

Add named styled tables:

```ruby
worksheet.append_row(["Code", "Status"])
worksheet.append_row(["0101000000", "Active"])
worksheet.add_table(
  0,
  0,
  1,
  1,
  name: "Commodity_watch_list",
  style: "TableStyleLight15",
  columns: ["Code", "Status"]
)
```

## Column Auto Width

Column auto width tracks string values and applies widths when the workbook
closes. It does not infer widths for numeric formats.

```ruby
workbook = FastExcel.open(constant_memory: true)
worksheet = workbook.add_worksheet
worksheet.auto_width = true
worksheet.append_row(["some text", "some longer text for example"])
workbook.close
```

## Low-Level libxlsxwriter API

This gem exposes FFI bindings for libxlsxwriter. You can call wrapped
libxlsxwriter functions directly:

```ruby
Libxlsxwriter.worksheet_activate(worksheet)
worksheet.activate
```

See the [libxlsxwriter documentation](https://libxlsxwriter.github.io/) for the
underlying C API.

## Development

Install dependencies and build the bundled libxlsxwriter shared library:

```sh
bundle install
make
make install
```

Run the test suite and examples:

```sh
bundle exec rake test
bundle exec rake examples
gem build uber_fast_excel.gemspec
```

On systems where zlib headers are not globally available, provide zlib include
and library paths when running `make`.

## Performance Validation

The gem is intended for fast, low-memory XLSX generation. CI runs a conservative
regression check that writes 20,000 rows in constant-memory mode on Ubuntu and
fails if it takes 10 seconds or longer:

```sh
bundle exec rake perf:validate
```

You can tune the local validation budget with environment variables:

```sh
PERFORMANCE_ROWS=50000 PERFORMANCE_MAX_SECONDS=20 bundle exec rake perf:validate
```

## Releases

Commit subjects are checked with [Cocogitto](https://docs.cocogitto.io/) in CI.
Use conventional commit subjects such as:

```text
feat: add rich string support
fix(workbook): close temporary files
ci: check conventional commits
```

`cog.toml` configures changelog generation from conventional commits and uses
`v`-prefixed release tags. Publishing is handled by the release workflow when a
`v*` tag is pushed; it builds `uber_fast_excel.gemspec` and publishes through
RubyGems trusted publishing.

Before the first release, create a pending trusted publisher in RubyGems:

- Gem name: `uber_fast_excel`
- Repository: `willfish/fast_excel`
- Workflow: `.github/workflows/release.yml`
